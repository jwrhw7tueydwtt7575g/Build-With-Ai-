================================================================================
🚀 AETHER FLOW: END-TO-END TECHNICAL DEEP DIVE & ARCHITECTURE 🚀
================================================================================

AETHER FLOW is an autonomous, 11-stage enterprise AI data pipeline that 
transforms raw, noisy datasets into executive-level strategy. It is built 
using a "Pipeline-as-a-Service" model where GPT-4o acts as the primary 
architect, dynamically selecting cleaning rules and enrichment methods 
based on real-time data profiling.

--------------------------------------------------------------------------------
1. INFRASTRUCTURE & PROVISIONING (Terraform)
--------------------------------------------------------------------------------
Every resource is provisioned via Terraform to ensure reproducibility.
- AZURE RESOURCE GROUP: A unified logical container for all services.
- ADLS GEN2 (Azure Data Lake Storage): Provides a multi-protocol, hierarchical 
  namespace. We use 'Landing' and 'Curated' zones for data isolation.
- AZURE SYNAPSE WORKSPACE: The unified analytics hub for data integration, 
  enterprise warehousing, and big data analytics.
- MANAGED IDENTITIES: A critical security feature. We use System-Assigned 
  Managed Identities so Synapse can access Storage and Key Vault without 
  storing hardcoded SAS tokens or passwords.
- AZURE KEY VAULT: Securely stores Azure OpenAI API keys and SQL credentials.
- AZURE OPENAI (GPT-4o): The "brain" of the pipeline, providing reasoning, 
  schema inference, and conversational intelligence.

--------------------------------------------------------------------------------
2. STAGE-BY-STAGE PIPELINE FLOW (The 11 Steps)
--------------------------------------------------------------------------------

STEP 1: INGESTION (FastAPI / ADLS Gen2)
- Service: FastAPI + Blob Webhook.
- Why: Provides an asynchronous entry point for large files. Data is landed 
  as 'Raw' in ADLS Gen2 for full traceability.

STEP 2: CANONICAL NORMALIZATION (GPT-4o)
- Service: Azure OpenAI (gpt-4o).
- Why: GPT-4o analyzes the first 50 rows of any unknown dataset to create a 
  Generic Schema mapping. It intelligently maps columns like "temp_C" or 
  "Degrees" to a standard name.

STEP 3: DELTA LAKE CONVERSION (Synapse Spark)
- Service: Apache Spark on Azure Synapse.
- Why: Converts normalized JSON/CSV into Delta Tables. This provides ACID 
  transactions, time-travel capabilities, and massive performance boosts for 
  downstream analytics.

STEP 4: STATISTICAL DIGEST (Pandas / Numpy)
- Why: Automated profiling. The system calculates variance, null counts, 
  uniqueness, and skewness for every field. This digest is the "input" for 
  the AI's planning logic.

STEP 5: LLM EXECUTION PLANNING (GPT-4o)
- Why: Autonomous Orchestration. GPT-4o reads the Step 4 Digest and 
  decides: "This field has 5% nulls, use Rule X," and "This field is free-text, 
  enrich with Sentiment Analysis." It outputs a JSON execution plan.

STEP 6: SPARK CLEANING EXECUTION (Custom Python Wrapper)
- Why: Reliability. Programmatic application of the AI plan. It handles 
  null-removal, duplicate deduplication, and type casting reliably across 
  millions of rows.

STEP 7: AI LANGUAGE ENRICHMENT (Azure AI Language Integration)
- Service: Azure Cognitive AI Language (Sentiment/NER/Urgency).
- Detail: Adds semantic depth. For every enriched field, we generate a 
  "Justification" and "Importance" score, explaining WHY the AI added it.

STEP 8: PATTERN & ANOMALY DISCOVERY (Scikit-Learn / IQR)
- Service: IQR Algorithm + Isolation Forest (Simulated).
- Why: Accuracy. Uses the Interquartile Range (IQR) method to find outliers. 
  Statistical patterns (correlations) are surfaced for business insights.

STEP 9: EXECUTIVE INSIGHT SYNTHESIS (GPT-4o)
- Why: Business Value. Translates raw statistical patterns (Step 8) into 
  human-readable takeaways like "Decrease in demand correlated with 
  Weather:Raining."

STEP 10: DYNAMIC NLQ SUGGESTION ENGINE (GPT-4o)
- Why: User Engagement. The AI analyzes the cleaning metrics and insights 
  to suggest 5 natural language questions the user SHOULD ask (e.g., 
  "Why are there nulls in Temperature?").

STEP 11: CONVERSATIONAL INTELLIGENCE HUB (Frontend Chatbot)
- Why: Grounded Q&A. A RAG-inspired chatbot that answers user questions 
  using the full context of Step 1 to Step 10. It cannot hallucinate 
  because it is "grounded" in the live dataset state.
